## 8.1 语音模式

### 8.1.1 /voice 命令

启用实时语音交互：

```bash
# 启用语音模式
/voice

# 退出语音模式
/exit-voice
```

### 8.1.2 实时语音交互流程

```
用户说话 → STT (Whisper) → 文本 → Agent 处理 → 响应 → TTS → 用户听到
```

### 8.1.3 TTS 引擎配置

| 引擎 | 配置项 | 说明 |
|------|--------|------|
| Edge (默认) | 免费、速度快 | `edge_voice: zh-CN-XiaoxiaoNeural` |
| ElevenLabs | 需要 API Key、音质好 | `voice_id: alex` |
| OpenAI | 需要 API Key | `model: tts-1, voice: alloy` |
| xAI | 需要 API Key | `voice: crystal` |
| Mistral | 需要 API Key | `voice: alloy` |

```yaml
# config.yaml
voice:
  enabled: true
  tts_engine: edge  # edge, elevenlabs, openai, xai, mistral

  tts:
    edge_voice: zh-CN-XiaoxiaoNeural
    speed: 1.0
    pitch: 0
```

### 8.1.4 语音转写（Whisper）

```yaml
voice:
  stt_engine: whisper

  stt:
    model: base  # tiny, base, small, medium, large
    language: auto
    whisper:
      model_path: ~/.hermes/models/whisper
```

使用本地模型：
```yaml
whisper:
  local_model: ~/.hermes/models/whisper/bin/googletts.onnx
```

### 8.1.5 语音频道集成（Discord 语音）

```yaml
voice:
  voice_channel:
    enabled: true
    platform: discord

    discord:
      auto_join: true  # 自动加入语音频道
      deafen: false    # 保持听觉
```

## 8.2 API 服务器

### 8.2.1 OpenAI 兼容 API

Hermes 暴露与 OpenAI 兼容的 API：


## 8.3 MCP 集成

### 8.3.1 MCP 协议概念

Model Context Protocol (MCP) 是 Anthropic 提出的工具协议标准：

```
┌─────────────┐     MCP      ┌─────────────┐
│   Client    │◄────────────►│   Server    │
│  (Hermes)   │              │ (Filesystem │
│             │              │  Git, etc.) │
└─────────────┘              └─────────────┘
```

核心概念：
- **Resources**: 文件、数据等可读资源
- **Tools**: 可执行的操作
- **Prompts**: 预定义的提示模板

### 8.3.2 MCP 服务器配置

```yaml
# config.yaml
mcp:
  enabled: true

  servers:
    # 文件系统服务器
    filesystem:
      command: npx
      args: ["-y", "@modelcontextprotocol/server-filesystem", "~/documents"]
      env: {}

    # Git 服务器
    git:
      command: uvx
      args: ["mcp-server-git", "--repository", "."]
      env: {}

    # GitHub 服务器
    github:
      command: npx
      args: ["-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_PERSONAL_ACCESS_TOKEN: ${GITHUB_TOKEN}

    # Slack 服务器
    slack:
      command: npx
      args: ["-y", "@modelcontextprotocol/server-slack"]
      env:
        SLACK_BOT_TOKEN: ${SLACK_BOT_TOKEN}
        SLACK_TEAM_ID: ${SLACK_TEAM_ID}

    # Brave 搜索
    brave-search:
      command: npx
      args: ["-y", "@modelcontextprotocol/server-brave-search"]
      env:
        BRAVE_API_KEY: ${BRAVE_API_KEY}
```

### 8.3.3 MCP OAuth 管理

```yaml
mcp:
  oauth_enabled: true

  oauth_clients:
    github:
      client_id: ${GITHUB_CLIENT_ID}
      client_secret: ${GITHUB_CLIENT_SECRET}
      auth_url: https://github.com/login/oauth/authorize
      token_url: https://github.com/login/oauth/access_token

    google:
      client_id: ${GOOGLE_CLIENT_ID}
      client_secret: ${GOOGLE_CLIENT_SECRET}
      auth_url: https://accounts.google.com/o/oauth2/v2/auth
      token_url: https://oauth2.googleapis.com/token
```

OAuth 授权流程：
```bash
# 触发授权
hermes mcp authorize github

# 浏览器打开授权页面
# 授权后自动保存 token
```

### 8.3.4 FastMCP 开发

使用 FastMCP 快速开发 MCP 服务器：

```python
# server.py
from fastmcp import FastMCP

mcp = FastMCP("My Server")

@mcp.tool()
def search_docs(query: str) -> str:
    """搜索文档"""
    return f"搜索结果: {query}"

@mcp.resource("docs://{path}")
def get_doc(path: str) -> str:
    """获取文档内容"""
    with open(path) as f:
        return f.read()

mcp.run()
```

运行自定义服务器：
```yaml
mcp:
  servers:
    my-server:
      command: python
      args: ["/path/to/server.py"]
      env:
        DEBUG: "true"
```

## 8.4 RL 训练

### 8.4.1 Tinker-Atropos 集成

Hermes 与 Tinker-Atropos RL 系统集成：

```bash
# 启动 RL 训练界面
hermes rl

# 查看可用环境
hermes rl environments
```

### 8.4.2 轨迹生成（Batch Runner）

```bash
# 生成训练轨迹
python batch_runner.py \
  --task "Solve this coding problem" \
  --model claude-opus-4 \
  --num-runs 100
```

配置：
```yaml
rl:
  batch_runner:
    tasks: ~/.hermes/rl/tasks.json
    output: ~/.hermes/rl/trajectories
    num_workers: 8

  # 评估
  evaluation:
    metrics:
      - pass_rate
      - solution_quality
      - tool_usage_efficiency
```
